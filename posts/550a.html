程设 C++ 上机模板速查手册

发表于2026-07-01|更新于2026-07-01|计算机
|浏览量:

用法：考试补充文件时，先看 main.cpp 调用了什么、.h/.hpp 要求什么签名，

再从本手册找对应模块，改类名/成员名直接套。所有代码都按你们课程风格写：

#ifndef 头文件保护 + using namespace std; + 声明(.h)与定义(.cpp)分离。

0. 每题开写前的 4 句自问（防漏条件，30 秒）

  1. main.cpp 到底调用了哪些函数？名字、参数、返回值、const 抄下来。

  2. .h/.hpp 里已经给了哪些成员变量和函数签名？签名一个字都不能改。

  3. 输入有没有：多组数据 / 空串 / 0 / 负数 / 越界 / EOF 结束？

  4. 输出格式：有没有空格、换行、大小写、保留几位小数、排序方向？

记住考试目标：6 题稳过 > 7 题都碰。一题卡 10 分钟不知补什么就跳，卡 20 分钟编译不过就跳。

1. 万能开头（背下来，任何题第一件事）

分文件规则（补充文件题最常见坑）：

  • .h/.hpp 只放声明，最外层永远包一层保护：

  • .cpp 里 #include "对应的.h"，成员函数一律写成 类名::函数名。

  • 模板类是唯一例外：模板的实现要写在 .h/.hpp 里（见第 8 节），不要单独放 .cpp，否则链接错误。

2. 普通类：声明 + 定义分离（周3–周4 基础）

Xxx.cpp

易错点清单：

  • 类声明 } 后面必须有分号 };。

  • .cpp 里成员函数必须 Xxx::，忘了就变成全局函数 → 一堆报错。

  • 声明是 const 成员函数，定义也必须带 const，两边要完全一致。

  • 构造优先用初始化列表；const 成员和引用成员只能用初始化列表。

3. 构造/析构/静态成员/常量（周4）

  • 默认参数只写在声明处：Xxx(int n, int v = 0);，定义处不要再写 = 0。

  • 无参能构造就提供默认构造，main 里 Xxx arr[10]; 才不报错。

6. 继承（周10–周11）

要点：

  • 派生类构造必须在初始化列表调用基类构造：: Base(参数)。

  • 构造顺序：先基类 → 再派生类成员；析构相反。

  • protected 成员：派生类能用，外部不能。

  • 想调用被隐藏的基类同名函数：Base::func();。

7.3 基类指针数组统一管理（多态最常见的题型）

多态题固定自查：

  • 基类析构 virtual 了吗？（漏了 → 内存泄漏 / 扣分）

  • 重写函数签名和基类完全一致吗？（参数、const 差一个都不算重写）→ 加 override 让编译器帮你查。

  • 抽象类的纯虚函数在派生类里全部实现了吗？没实现完 → 派生类还是抽象类，new 会报错。

  • 用基类指针/引用调用才有多态；用对象值调用会“对象切片”，退化成基类。

12. 高频编译错误 → 秒修对照表

| 报错现象 | 原因 | 修法 |

|—|—|—|

| expected ';' after class definition | 类 } 后漏分号 | 加 }; |

| undefined reference to 'Xxx::f()' | 成员函数没定义 / 静态成员没类外定义 / 模板实现放了 .cpp | 补定义；模板实现搬进 .h |

| 'xxx' was not declared | 忘 include / 拼写错 / 忘 using namespace std | 补头文件 |

| passing 'const Xxx' as this discards qualifiers | const 对象调了非 const 函数 | 给读取类函数加 const |

| no match for 'operator<' | sort/优先队列缺比较 | 补 operator< 或传 lambda |

| cannot allocate abstract type | 抽象类纯虚没实现全 | 派生类实现所有 =0 函数 |

| invalid use of incomplete type | 头文件互相 include | 用前向声明 class X; |

| 运行崩 / double free | 浅拷贝 | 补深拷贝三件套（第4节） |

13. 考场 180 分钟节奏（贴墙上）

| 时间 | 动作 |

|—|—|

| 0–5min | 扫 7 题，每题标 A(会)/B(可能)/C(时间杀手)，定顺序 |

| 5–65min | 先做 3 道 A：字符串/基础类/STL |

| 65–125min | 2 道中等：深拷贝 / 多态 / 优先队列 |

| 125–160min | 第 6 题：模板或较熟的模拟题 |

| 160–175min | 回收一个半成品，至少让它编译过 + 过样例 1 |

| 175–180min | 全部重编译，确认没漏交文件 |

跳题红线：读完 5 分钟不知补什么 → 跳；写 15 分钟全红 → 跳；要推倒重写 → 先交能编译版本，最后回来。目标是 6 题有效提交，不是 7 题完美。

附：每种题型“认脸”速查

| 看到这些关键词 | 用哪节模板 |

|—|—|

| 分数/复数/坐标/时间 + + - * < == | 第 5 节 运算符重载 |

| int* / char*、new、深拷贝、MyString、Matrix | 第 4 节 三件套 |

| virtual、基类指针数组、Shape/Animal/Vehicle | 第 7 节 多态 |

| = 0、抽象、接口、device/cargo | 第 7.2 抽象类 |

| template、<T>、<T, N>、Filter/Manager | 第 8 节 模板 |

| 优先队列、排队、分诊、Top-K、括号匹配 | 第 9 节 STL |

| 命令 push/pop/query 循环 | 第 10 节 命令分发 |
