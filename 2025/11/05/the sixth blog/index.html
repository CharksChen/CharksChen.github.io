程序设计期中知识汇总

根据11月4日视频课分享内容，将前半学期所学知识进行汇总，以供复习

第一部分：编程知识

输入

整型数

short %hd -215215-1
int %d -231231-1
long long %lld -263263-1
unsigned int %u 0232-1
unsigned long long %llu 0~264-1

浮点数

float %f 6位小数
double %lf 15位小数
long double %llf 18位小数

字符

char %c
string %s

输出

除此之外还要特别注意的有
%o 无符号八进制整数，不输出前缀0
%u 无符号十进制整数
%x/X 无符号十六进制整数，不输出前缀0x,字母有大小写区分
%e/E 科学计数法，e有大小写区分
%g/G 有效位数，如%.8g保留8位有效数字（自动简化）
%p 以十六进制形式输出指针
%n 不读取任何字符，只读取截至目前读取的字符个数

其他

常量 const
转义字符\n \"
存储类 static extern

运算符

位运算（整型、字符型）

& 与
| 或
~非
^异或
<<左移
>>右移

三目运算符

a > b ? a : b

类型转换

隐式类型转换

显示/强制类型转换

报错&调试

3.理解编译错误

  1. AC - Accepted（答案正确）
  2. WA - Wrong Answer（答案错误）
  3. TLE - Time Limit Exceeded（超出时间限制）
  4. RE - Runtime Error（运行时错误）
  5. CE - Compilation Error（编译错误）
  6. MLE - Memory Limit Exceeded（超出内存限制）
  7. OLE - Output Limit Exceeded（输出限制超出）
  8. PE - Presentation Error（格式错误）
  9. TCI - Time Complexity Incorrect（时间复杂度错误）
  10. MCI - Memory Complexity Incorrect（空间复杂度错误）

正确开数组

一般要在数据范围之外多开5。
一般建议开成全局数组，开在主函数里有爆栈的风险。在编译选项里，编译时加入以下命令-wl,--stack,268435456.
同时，单个数组大小不要超过107,不然会编译错误。
